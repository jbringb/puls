# Security Policy

## Reporting a Vulnerability

If you find a security issue in Puls, please report it privately via
[GitHub Security Advisories](https://github.com/jbringb/puls/security/advisories/new)
rather than opening a public issue.

Include what you found, the affected version/commit, and steps to reproduce.
I'll acknowledge reports within a few days and aim to ship a fix before any
public disclosure.

## Supported Versions

Puls does not yet follow semantic versioning; only the `main` branch is
supported. Please report issues against the latest commit.
